# AGENTS.md - 数学物理方法课程讲义

## 项目定位

本项目是一份中文 LaTeX 讲义：《数学物理方法》。目标不是写成只给结论的教材，而是写成一份真正适合本科生，尤其是基础较弱学生自学的讲义。

默认读者：
- 大二下学期本科生
- 已学过高等数学、大学物理
- 基础不一定扎实，但愿意认真学

## 核心写作原则

1. **自学友好优先**：默认学生容易卡在抽象定义、长推导、边界条件和章节联系上。宁可讲慢一点、细一点，也不要跳关键台阶。
2. **先讲为什么，再讲是什么**：每个概念、方法、定理都应先说明它为什么出现、解决什么问题、和前面内容有什么联系，再给出正式定义和推导。
3. **口语化但严谨**：写作要像老师上课，通俗、自然、循循善诱，尽量让学生愿意读下去；但数学表达必须严格。
4. **强调来龙去脉**：学生不能只看到结论，必须知道对象从哪里来、条件为什么要有、方法为什么有效、结果有什么物理意义。
5. **物理主线明确**：始终围绕“物理问题 → 数学模型 → 数学求解 → 物理解释”组织内容。
6. **图文并茂**：尽量多配图，图是教学工具，不是装饰。优先使用 TikZ。

## 章节默认结构

每章、每节尽量遵循：

物理背景/引子 → 定义与基本概念 → 定理与证明 → 方法与推导 → 例题与完整解答 → 小结 → 习题

写节内容时默认包含：
- 物理背景或问题引入
- 正式定义、定理和证明
- 关键推导，不跳关键步
- 完整例题与解答
- 小结
- 分层习题：基础题、综合题、挑战题

## 内容要求

1. 每节尽量从物理问题出发，而不是直接空降定义。
2. 所有重要定理必须给出证明，证明应分步骤展开。
3. 例题必须给出完整解答，使用 `jie` 环境。
4. 主动提醒易错点、常见误区、与前修课程的联系。
5. 主动指出不同章节之间的联系，避免内容割裂。
6. 遇到学生可能卡壳的地方，要主动解释“为什么这一步可以这样做”。

## 图示要求

应尽量为以下内容配图：
- 复平面
- 积分路径
- 边界条件
- 驻波与振动模式
- 坐标系
- 函数图像
- 球对称/柱对称问题
- 变换与展开的几何直观

图示要求：
- 优先用 TikZ
- 清晰、简洁、服务理解
- 文字说明少而准
- 使用 `figure`、`caption`、`label`

## LaTeX 规范

1. 所有 `.tex` 文件使用 UTF-8 编码。
2. 所有数学符号必须放在数学环境中。
3. 禁止在文本模式中直接使用 Unicode 数学符号。
4. 中文引号统一使用 LaTeX 形式：``...''。
5. 不在章节文件中定义新环境；所有环境统一在主文件中定义。
6. 输出内容应可直接放入项目的章节文件中。

## 项目常用环境

- `dingli`：定理
- `dingliyi`：定义
- `yinli`：引理
- `tuilun`：推论
- `lizi`：例题
- `zhu`：注
- `sikao`：思考题
- `jie`：解答
- `jinshi`：易错点/警示
- `tishi`：提示/学习建议/前修知识回顾
- `wulibj`：物理背景/导读

## 编译与检查

推荐编译：

```bash
xelatex main.tex
xelatex main.tex
```

错误检查：

```bash
xelatex -interaction=nonstopmode main.tex
grep -i "error" main.log
```
